# Antigravity Compass for ApplyFlow

> This document serves as the **guiding compass for Antigravity** when generating or assisting with code in ApplyFlow.  
> It combines usage principles, strict guidelines, and a pre-generation enforcement checklist to ensure accuracy, security, scalability, and adherence to PRD and user flows.

---

## 1. Philosophy

1. **AI is an Assistant, Not the Driver**
   - Antigravity accelerates tasks but **does not make decisions**.  
   - Developer owns **architecture, core logic, and flow decisions**. AI executes only within defined boundaries.

2. **Precision Over Speed**
   - All AI-generated code must be **manually verified**.  
   - Speed is allowed only when correctness, security, and scalability are ensured.

3. **Define the “What” Before the “How”**
   - PRD, user flows, database schema, and API contracts **must exist first**.  
   - Antigravity implements **pre-approved specifications only**.

4. **Non-Negotiable Security & Performance**
   - No bypassing of authentication, authorization, or RLS.  
   - Input must always be sanitized and validated.  
   - AI code must scale and perform efficiently.

---

## 2. Allowed AI Uses

- Generate boilerplate code (components, hooks, Edge Functions scaffolding).  
- Scaffold structured templates based on **PRD and user flows**.  
- Write unit tests for verified logic.  
- Refactor repetitive code **without changing behavior**.  
- Format or organize code for readability.  

> Always include comments and explanations in AI-generated code.

---

## 3. Forbidden AI Uses

1. Architectural or flow decisions.  
2. Implementing features without prior **PRD/user flow approval**.  
3. Bypassing authentication, validation, or RLS.  
4. Generating code that **assumes missing requirements**.  
5. Guessing or improvising feature logic.  
6. Introducing insecure patterns or data leaks.

---

## 4. Security & Privacy Protocols

- Ensure **authentication & authorization** for user actions.  
- Enforce **RLS policies** for database queries.  
- Validate and sanitize all inputs: strings, numbers, files, URLs.  
- Edge Functions must be **secure and resilient**.  
- AI-generated code cannot expose secrets.

---

## 5. Performance & Scalability Rules

- Check loops, queries, and API calls for efficiency.  
- Use pagination, batching, or caching where needed.  
- Avoid unbounded operations that could degrade performance.  
- Optimize AI calls (resume parsing, job match, interview prep) for size and complexity.  

---

## 6. Documentation Requirements

- All AI-generated code **must include comments** explaining:
  - Purpose  
  - Inputs and outputs  
  - Related PRD, flow, or DB schema references  
- Tag AI-generated code clearly:  
  `// Generated by Antigravity, reviewed per AG Compass`

---

## 7. Pre-Generation Enforcement Checklist

Before any AI code generation, Antigravity must verify:

### 7.1 Feature Verification
- Feature exists in PRD or user flow document.  
- Inputs, outputs, and acceptance criteria are defined.  
- Dependencies (tables, APIs, state) are known.

### 7.2 Security Compliance Check
- Authentication enforced.  
- Authorization/RLS policies verified.  
- Inputs sanitized and validated.  
- Secrets are not exposed.

### 7.3 Performance & Scalability Check
- Loops, queries, API calls efficient.  
- Pagination or batching for large datasets.  
- AI calls optimized and trimmed.  
- No unbounded rendering or memory-heavy operations.

### 7.4 Flow & UX Check
- User flow strictly followed:  
  **Login → Add Job → Resume Upload → AI Tailoring → Interview Prep → Track Progress → Dashboard**  
- One primary action per screen.  
- Data propagation between screens is consistent.

### 7.5 Documentation & Comments
- Purpose, inputs, outputs explained.  
- PRD or flow references included.

### 7.6 Testing Requirements
- Unit or integration tests defined.  
- Tests cover success, edge cases, and errors.

### 7.7 Final Decision
- ✅ All checks passed → generate code.  
- ❌ Any check failed → halt and alert developer.

---

## 8. Enforcement Philosophy

- **No guessing:** AI stops if any requirement is unclear.  
- **Accuracy first, speed second:** Performance and correctness are prioritized.  
- **Flow fidelity:** User experience and PRD alignment are mandatory.  
- **Security, scalability, and maintainability** cannot be compromised.
